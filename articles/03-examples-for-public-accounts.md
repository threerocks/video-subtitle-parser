# 用一个视频链接生产两种公众号文章：国漫解析和情绪写作的共同底层

两个公众号，看起来可以完全不同。

「一只国漫观察猫」写国漫解析。它关心人物、剧情、伏笔、镜头和改编。

「人间缓冲站」写情绪和关系。它关心一个人为什么会卡住，为什么反复内耗，为什么明明知道道理，仍然走不出来。

表面上，这是两种内容。

但底层有一个共同问题：素材要先变成可以复查的文本。

拿国漫解析来说，一条 B 站视频可能叫“阮秀人物志，只爱陈平安的火神转世，书友心中的意难平”。这不是逐帧解析，也不是第 25 集解析。它是人物志。

如果 AI 只看到目录名里有 25，或者看到项目过去都在写逐帧解析，就很容易顺手套一个标题：

```text
《剑来2》第25集逐帧解析，阮秀人物志与火神转世的一场人间贪恋
```

这个标题看起来像公众号标题，实际是错的。

错不在文字润色，而在素材类型判断。源视频是人物志，文章就应该是人物志。不能因为它被放进了一个系列目录，就硬追加“第几集”和“逐帧解析”。

Video Subtitle Parser 在这里提供的不是文案答案，而是判断依据。

它会把源视频标题写进 metadata，把文字内容清理出来。后续 agent 在写作前，应该先看 metadata，再看 transcript。这样它就能知道：这是阮秀专题，不是集数逐帧。标题应该围绕阮秀，而不是围绕第 25 集。

同样的逻辑放到「人间缓冲站」也成立。

假设你拿到一个 YouTube 长视频，主题是亲密关系里的情绪反复。AI 很容易把它写成泛泛的心理学鸡汤：边界感、原生家庭、依恋类型、向内探索。每个词都像对的，但可能都不是视频里真正讲的。

这时本地文字稿就是刹车。

你可以要求 agent：只使用 `*_transcript_turbo_clean.txt` 里的信息。视频里有一个故事，就围绕那个故事；视频里没有提某个理论，就不要硬塞理论。文章不是显得懂很多，而是把材料里已有的东西讲清楚。

这就是两个公众号共享的底层方法：

先让视频落地，再让 AI 写作。

实际流程很短。

```bash
video-subtitle-parser "$URL" \
  --platform auto \
  --out-dir materials/video-001
```

如果是《剑来》这种专名很多的内容，再加词表：

```bash
video-subtitle-parser "$URL" \
  --term-file examples/jianlai_terms.txt \
  --initial-prompt "陈平安 宁姚 阮秀 骊珠洞天 落魄山" \
  --out-dir materials/jianlai-video-001
```

然后让 AI 做三件事：

第一，看 metadata，判断素材类型。

第二，看 cleaned transcript，拆文章结构。

第三，看 segments，在需要时回查时间点。

这套流程并不炫。

它像一个内容生产里的地基。地基看不见，但没有它，后面所有漂亮句子都可能歪。

国漫解析也好，情绪写作也好，真正有价值的不是 AI 生成速度，而是它有没有忠实地站在材料上。Video Subtitle Parser 把材料递到它面前。

剩下的，才是写作。

如果你想看这套流程在国漫解析里的用法，可以关注「一只国漫观察猫」。如果你想看它如何进入情绪和关系类写作，可以关注「人间缓冲站」。两个号题材不同，但都会越来越依赖同一件事：让内容先有证据，再有表达。
